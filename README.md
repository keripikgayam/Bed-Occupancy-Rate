🏥 Bed Occupancy Rate (BOR) Analysis & Dashboard
📌 Project Overview
    This project focuses on analyzing the Bed Occupancy Rate (BOR), a critical Key Performance Indicator (KPI) in healthcare operational management. Maintaining an optimal bed utilization rate is essential for ensuring service efficiency, avoiding hospital overcapacity, and stabilizing revenue streams.
    This repository contains an end-to-end data analytics workflow—from raw data processing and Exploratory Data Analysis (EDA) to the development of an interactive dashboard designed to support data-driven decision-making for operations and management teams.

🎯 Business Objectives
- Capacity Planning: Identify daily and monthly trends in bed utilization to optimize the allocation of medical staff and resources.
- Operational Efficiency: Analyze the correlation between BOR, patient Length of Stay (LOS), and overall operational bottlenecks.
- Data-Driven Strategy: Provide clear, actionable metrics to executive stakeholders to minimize operational risks during peak patient surges.

🛠️ Tools & Technologies Used
- Python (Pandas, NumPy): Utilized for data cleaning, transformation, and feature engineering on patient admission datasets.
- SQL: Applied for querying, aggregating, and validating operational metrics from simulated relational databases.
- Data Visualization (Tableau / Matplotlib / Seaborn): Used to design intuitive and interactive dashboards for regional or executive-level reporting.

📊 Key Features & Analysis Steps
1. Data Preprocessing: Handled missing values, formatted admission/discharge timestamps, and calculated accurate Length of Stay (LOS) metrics.
2. Exploratory Data Analysis (EDA): Analyzed patient distribution across different wards, room types, and seasonal timeframes.
3. Dashboard & Reporting: Developed visualizations tracking historical BOR fluctuations, turnover intervals, and available capacity.
